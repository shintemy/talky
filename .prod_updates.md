# Talky 产品与研发进展更新（2026-03-15）

## 1. 重要背景（产品定位与核心逻辑）

- 产品定位：`Talky` 是面向 macOS（Apple Silicon）的本地语音输入助手，主打本地优先（ASR + LLM 都在本机或局域网可控节点）。
- 目标场景：高频开发与写作输入，强调隐私（不上传云端）、文本可用性（自动清理口语冗余）、跨应用实用性（自动粘贴 + 兜底浮窗）。
- 核心交互：按住热键录音 -> 松开触发转写（ASR）-> 本地大模型清理（LLM）-> 自动粘贴到当前焦点应用（无焦点时显示复制面板）。
- 架构主线（代码实现对应）：
  - 控制层：`main.py` + `talky/controller.py` 负责生命周期、热键流程、错误提示、权限引导。
  - ASR：`talky/asr_service.py`（Whisper）。
  - LLM 清理：`talky/llm_service.py`（Ollama）。
  - 输入输出：`talky/recorder.py`（录音）、`talky/paster.py`（粘贴）、`talky/focus.py`（焦点检测）。
  - 配置与历史：`talky/config_store.py`、`talky/history_store.py`（按天写入 `history/YYYY-MM-DD.md`）。
- 启动策略：`start_talky.command` 已具备自动更新、依赖检测/安装、Ollama 本地/局域网模式切换、首次引导配置等逻辑。

## 2. 当前开发进度（已完成到什么程度）

- 基础产品链路已落地：从“按住说话”到“清理文本并输出”的闭环已具备。
- 本地化能力已落地：
  - 本地模型下载脚本存在（`download_model.py`）。
  - 启动脚本会校验依赖、检查/拉起 Ollama（本地模式）、解析模型并启动主程序。
- 稳定性增强已具备：
  - 录音、处理流程中有较完整的异常捕获与状态管理（例如防重复输出、短文本跳过 LLM）。
  - 启动与运行期均有错误提示路径（权限、Ollama 可用性、模型缺失等）。
- 可维护性基础较好：
  - 已有较多单元测试文件（配置、词典修正、历史存储、文本防护、权限等模块）。
  - 中英文 README 已覆盖安装、首启、日常使用与常见故障排查。

## 3. 待解决 Bug / 风险项（当前可确认）

### A. 测试环境导入失败（当前最直接阻塞）

- 现象：在仓库根目录执行 `pytest -q` 时，测试收集阶段出现 `ModuleNotFoundError: No module named 'talky'`，导致多项测试无法运行。
- 影响：当前 CI/本地回归无法作为质量门禁，阻塞后续迭代速度与发布信心。
- 初步判断：
  - 更偏向“测试运行方式/环境配置”问题，而非业务功能本身立即崩坏。
  - 需统一测试入口（例如 `python -m pytest`）并补充包路径或安装方式约束（editable install 或标准化 `PYTHONPATH`）。

### B. 运行层已知高频问题（文档已记录，仍属于用户侧常见故障）

- Whisper 本地模型下载中断/损坏：需要删除 `local_whisper_model` 后重新下载。
- Ollama 预热 `502`：需要重启 Ollama 服务。
- 权限依赖敏感：麦克风 / 辅助功能授权缺失会影响核心体验（尤其自动粘贴）。
- 网络与代理依赖复杂：模型下载路径对代理配置较敏感，容易在首次安装阶段失败。

## 4. 接下来几步具体计划（建议按优先级执行）

1. **先修测试可执行性（P0）**
   - 固化唯一测试命令（推荐写入 README 与脚本）。
   - 修复 `talky` 包导入路径问题，确保 `pytest` 能稳定收集并运行。
   - 目标：先把当前已有测试跑通，建立最小可用回归门禁。

2. **补齐启动与安装的“可观测性”（P0）**
   - 明确首启失败原因分类（依赖缺失、模型缺失、Ollama 不可达、权限不足）。
   - 将关键错误提示统一为可执行动作（命令级提示），降低排障成本。

3. **收敛模型与主机配置体验（P1）**
   - 优化本地/局域网 Ollama 模式切换后的配置一致性与提示闭环。
   - 为常见网络环境（直连/HTTP 代理/SOCKS）整理一页“最小成功配置”。

4. **完善质量保障（P1）**
   - 为核心闭环补充端到端烟雾测试（热键触发可用 mocked 层替代）。
   - 增加最关键路径的回归检查：ASR 空文本、LLM 空输出、无焦点兜底、历史写入。

5. **推进发布节奏管理（P2）**
   - 维护本文件为每周/每版本更新日志（背景不变、进度/Bug/计划持续滚动更新）。
   - 与 `start_talky.command` 的自动更新策略配套，形成“可发布版本 + 可回滚”规范。
